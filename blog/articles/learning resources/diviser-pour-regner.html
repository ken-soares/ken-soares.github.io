1. Explication du paradigme "Diviser pour régner"

Le paradigme diviser pour régner est une méthode algorithmique basée sur le principe suivant: On prend un problème complexe que l'on divise en une multitude de petits problèmes. Une fois les petits problèmes résolus, on les recombine afin d'obtenir la solution au problème de départ.

1.1. TL;DR

On a donc 3 étapes:

  • Diviser: Diviser le problème en un certain nombre de sous-problèmes.
  • Régner: Résoudre les sous-problèmes
  • Combiner: On combine les solutions des sous-problèmes afin d'obtenir la solution du problème de départ

2. Application concrète avec le tri fusion (merge sort)

On va donc refaire en Python l'algorithme de tri fusion permettant de mieux comprendre le paradigme "diviser pour régner" 

# on se sert de cette fonction pour fusionner deux listes triées
def fusion(l1,l2):
	lvide = []
	while l1 != [] and l2 != []:
		if l1[0] < l2[0]:
			lvide.append(l1[0])
			l1 = l1[1:]
		else:
			lvide.append(l2[0])
			l2 = l2[1:]
	lvide = lvide + l1 + l2
	return lvide

# on trie une liste en divisant recursivement la liste de depart
# et en les recombinant, triées par la fonction fusion()
def tri_fusion(t):
	if len(t) <= 1:
		return t
	else:
		l1 = tri_fusion(t[:len(t) // 2])
		l2 = tri_fusion(t[len(t) // 2:])
	return fusion(l1,l2)

Cet algorithme de tri utilise la récursivité et permet de trier un tableau avec une complexité largement inférieure a celle du tri par selection par exemple.

complexité O(nlog₂n) O(n²)
